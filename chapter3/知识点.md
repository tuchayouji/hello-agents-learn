## ⭐ Transformer 核心组件

### 自注意力机制（Self-Attention）

**本质**：让模型在处理每个词时，能关注句子中所有其他词

```
Attention(Q,K,V) = softmax(QK^T / √dk) × V
```

| 角色 | 含义 |
|------|------|
| **Q (Query)** | 当前词，主动查询 |
| **K (Key)** | 可被查询的标签 |
| **V (Value)** | 词本身携带的信息 |

### 多头注意力
- 将 Q/K/V 切分成 h 份
- 每份独立计算注意力
- 捕捉不同类型的语义关系

## ⭐ Decoder-Only 架构 (GPT)

### 核心思想
- 完全抛弃编码器，只保留解码器
- **自回归**：预测下一个词

### 掩码自注意力
- 将未来位置的分数设为 -∞
- 防止"偷看"未来信息

### 为什么成功？
- 训练目标统一（预测下一个词）
- 结构简单，易于扩展
- 天然适合生成任务

### ⭐ 思维链 (CoT) 示例

```
问题：一个篮球队80场赢了60%，接下来15场赢了12场，总胜率？

思考：
1. 第一赛季赢了 80×60% = 48 场
2. 总比赛 80+15 = 95 场
3. 总胜利 48+12 = 60 场
4. 总胜率 60/95 ≈ 63.16%
```

## ⭐ 分词 (Tokenization)

### BPE 算法
1. 初始化：所有基本字符
2. 迭代合并：最高频相邻词元对
3. 重复直到词表达到预设大小

### 对开发者的意义
- 上下文窗口以 **Token** 计算
- API 按 **Token** 计费
- 影响模型表现（空格、大小写）
