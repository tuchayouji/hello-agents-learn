# 第五章 基于低代码平台的智能体搭建

> **一句话总结**：低代码平台让不会写代码的人也能搭建智能体，核心是"拖拽+配置"

---

## ⭐ 为什么需要低代码平台

| 痛点 | 低代码解决方案 |
|------|----------------|
| 技术门槛高 | 封装成可视化节点，拖拽连接 |
| 开发周期长 | 分钟级搭建原型 |
| 调试困难 | 可视化运行轨迹 |
| 协作不畅 | 标准化组件和最佳实践 |

---

## ⭐ 四大平台对比

| 平台 | 定位 | 核心优势 | 适用场景 |
|------|------|----------|----------|
| **Coze** | 零代码Agent构建 | 插件丰富、一键发布 | 快速验证、个人创作 |
| **Dify** | 开源LLM应用平台 | 企业级、RAG Pipeline | 专业开发、企业应用 |
| **FastGPT** | 知识库问答专精 | RAG优化、可视化Flow | 知识库客服、文档问答 |
| **n8n** | 通用工作流自动化 | 连接各类SaaS服务 | 业务流程集成 |

## ⭐ 实践案例：每日AI简报助手

### 构建步骤

1. **添加信息源插件**
   - RSS：订阅36氪、虎嗅、IT之家
   - GitHub：追踪AI开源项目
   - arXiv：获取最新论文

2. **配置插件参数**
   ```
   RSS: 36kr.com/feed, rss.huxiu.com
   GitHub: q=AI, per_page=10, sort=updated
   arXiv: count=5, search_query=AI
   ```

3. **设定角色与提示词**
   - 角色：资深科技媒体编辑
   - 输出：10条新闻 + 5篇论文 + 5个项目

4. **测试发布**
   - 一键发布到微信、飞书、豆包等

## ⭐ Coze 优势与局限

| ✅ 优势 | ❌ 局限 |
|---------|---------|
| 插件生态强大 | 不支持MCP协议 |
| 可视化编排直观 | 部分插件配置复杂 |
| 提示词控制灵活 | 无法导入JSON工作流 |
| 多平台部署便捷 | 付费版才能导出 |
