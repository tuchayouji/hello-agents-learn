# 第四章 智能体经典范式构建

> **一句话总结**：三种范式解决不同问题——ReAct 边想边做，Plan-and-Solve 先谋后动，Reflection 自我迭代

---

## ⭐ 三种范式对比

| 范式 | 核心思想 | 适用场景 | 灵活性 | 效率 |
|------|----------|----------|--------|------|
| **ReAct** | 边想边做 | 需要外部信息 | ⭐⭐⭐ | 多次调用 |
| **Plan-and-Solve** | 先规划后执行 | 结构化推理 | ⭐ | 规划1次+N次 |
| **Reflection** | 执行→反思→优化 | 需要高质量输出 | ⭐⭐ | 迭代优化 |

---

## ⭐ ReAct (Reason + Act)

### 核心循环
```
┌────────────────────────────────────────┐
│  Thought → Action → Observation → 循环  │
└────────────────────────────────────────┘
```

### 工作流程
1. **Thought**：分析问题，规划下一步
2. **Action**：调用工具或 `Finish[答案]`
3. **Observation**：获取工具返回结果
4. 重复直到得出最终答案

### ⭐ 工具定义三要素

| 要素 | 说明 | 重要性 |
|------|------|--------|
| **名称** | 简洁唯一标识符 | ⭐⭐⭐ |
| **描述** | 告诉LLM何时使用 | ⭐⭐⭐⭐⭐ |
| **执行逻辑** | 真正执行的函数 | ⭐⭐⭐ |

### 优缺点

| ✅ 优点 | ❌ 局限 |
|---------|---------|
| 高可解释性（Thought链） | 对LLM能力强依赖 |
| 动态规划与纠错 | 执行效率低（多次调用） |
| 工具协同能力 | 提示词脆弱性 |

---

## ⭐ Plan-and-Solve

### 两阶段流程
```
规划阶段 ──────────────► 执行阶段
    │                        │
生成计划(Python列表)      逐步执行计划
```

### 与 ReAct 对比

| 对比项 | ReAct | Plan-and-Solve |
|--------|-------|----------------|
| **决策方式** | 边想边做 | 先规划后执行 |
| **灵活性** | 高 | 低 |
| **适用场景** | 动态环境 | 结构化任务 |
| **LLM调用** | 每步都调用 | 规划1次+执行N次 |

---

## ⭐ Reflection

### 核心循环
```
执行(初稿) → 反思(评审) → 优化(修订) → 循环
```

### 价值
- **内部纠错**：不依赖外部反馈
- **持续优化**：从有缺陷到完善
- **短期记忆**：存储执行-反思轨迹
